# Technical Assessment: ReAct Agent Execution Engine

## Overview
Modern AI agents operate on the "ReAct" (Reason + Act) pattern. The LLM generates a block of text containing its internal reasoning and a specific command to trigger an external tool. 

Your objective is to build a `ToolRegistry` and an `AgentExecutor` in pure Python that parses a raw string from an LLM, securely executes the corresponding Python function, and returns the observation.

## Core Requirements

1. **Framework:** Pure Python (3.10+) and Pydantic.
2. **Allowed Modules:** `re`, `json`, `typing`, `callable`. (NO LangChain, NO OpenAI SDK, NO external agent frameworks).
3. **The Data Models & Registry:**
   * Create a Pydantic model `ToolContext` that stores a tool's `name`, `description`, and the actual Python `callable` (the function).
   * Create a `ToolRegistry` class with a method `register(name: str, description: str, func: callable)` that stores available tools.
4. **The Parsing & Execution Engine (`AgentExecutor`):**
   Create a class that takes a `ToolRegistry` upon initialization. It must implement:
   * `execute_llm_output(self, llm_output: str) -> str`: This method receives the raw text generated by the LLM. 
     * **Step 1:** It must parse the text to extract the `Action` (the tool name) and the `Action Input` (a JSON string representing kwargs).
     * **Step 2:** It must find the tool in the registry. If the tool does not exist, return exactly: `"Error: Tool '{name}' not found."`
     * **Step 3:** It must parse the JSON input. If the JSON is malformed, return exactly: `"Error: Invalid JSON input for tool."`
     * **Step 4:** It must execute the tool with the unpacked JSON arguments. If the tool executes successfully, return the output as a string formatted as: `"Observation: {result}"`. If the python function throws an exception, catch it and return: `"Error: Tool execution failed - {error_message}"`.
